When that is complete the zio package will support a couple more approaches to teaching Java.

  1. Start the class algorithmically with the InOut class or with the Input and Output classes. Continue with the ZBox, ZRow, ZCol, and ZWindow classes showing how colored boxes can be arranged in a window. Finish by replacing those colored boxes with subclasses of ZKeyboard ZPicture and ZLabel.

    All but the last step are possible with existing features of the zio package.

  2. Create an interactive class from the zio package. Start the class with objects first. Have students program by making subclasses of the interactive class. These subclasses will all have their own version of a processInput() method.

    Later show how the interactive class is implemented and let students create variations using the building blocks from the zio package.

    The classes necessary for this approach will soon be in the zio package. The teaching techniques required for making this approach work will take longer to develop. Anybody interested in helping?
