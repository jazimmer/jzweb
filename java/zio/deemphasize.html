- Deemphasize I/O -

author

The idea behind this approach is to emphasize basic data types and converting data from one type to another. Arrays are also introduced early. Not much is said about I/O.

For writing on the command window use Zio.show() and for reading and writing on files use InOut.readText() and InOut.writeText(). Example,

public static void main( String [] ignore ) {
     String theFile = InOut.readText("someFileName.txt");
     Zio.show(theFile); 
     InOut.writeFile("someFileName.txt","This material has been censored.");
}

which will read an entire file, dump it to the command window, and rewrite the file so that its entire contents is "This material has been censored."

This approach requires very little time to explain I/O. Later when students know something about arrays, they can use additional methods in the Zio class which take the Strings obtained with InOut.readText() and separate them into lines, characters, or tokens.

The use of Zio.show is unnecessary. You could use System.out.println() just as well. Two reasons to use Zio.show() are

  • It only writes Strings, not ints or doubles. This restriction forces students to remain aware of the differences between data types.
  • It exists in the same class as other useful methods and so using it minimizes the number of classes students must be aware of.

The InOut and Zio classes are freely available here.
