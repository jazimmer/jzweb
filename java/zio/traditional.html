- Traditional I/O -

author

The idea behind this approach is to teach a more traditional form of I/O as was done, for example, in the days of Pascal or C and to do so in a way that uses the same methods for user interaction as for file I/O.

Use the Input class for reading from the keyboard (via the command window) or from a text file and use the Output class for writing to the command window or to a text file.

Methods in these classes are not overloaded so students must keep track of which data types they are working with. This seems a bit heavy but it is good practice for the kinds of things a student needs to keep straight when taking the AP exam.

The Input class provides a unified way of reading from lots of sources and the Output class provides a unified way of writing to lots of destinations. More advanced students can use these classes for socket I/O or to replace the (now deprecated) StringTokenizer class. The main limitation is that these classes work only with text, not images or sound.

The Input and Output classes are freely available here.
