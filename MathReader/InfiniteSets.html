Sizing Sets

Are these two sets the same size?
One way to answer is to say “No, because the one on the left has three elements and the one on the right has four.”
Counting is an effective way to compare the sizes of some sets, but when you try to count some other sets, the counting never ends. We call those sets infinite. Finite sets of course are those that can be counted in finite time. To know a set is finite we must be able to show that counting will come to an end—although perhaps not in our lifetimes.
In this chapter we will look at a way to do endless counting. Later on we will consider the possibility that even endless counting is not enough to size some sets.
Of course we need examples of infinite sets to work with. Our number system provides plenty of examples.

Numbers Again

The real numbers R and two of its subsets, Z (the integers) and N (the natural numbers), are good subjects talking about things infinite because we have a common understanding about their properties. When talking about recursive functions we worked with intervals of real numbers. Here we will work with intervals of integers as well. To distinguish between different kinds of intervals we will use the following notation.
  • R[x,y] represents all the numbers between x and y including both x and y (x and y are numbers with x≤y)
  • Z[x,y] represents all the integers between x and y including both x and y (x and y are integers with x≤y)
  • R(x,y) represents all the numbers between x and y excluding both x and y (x and y are numbers with n<m)
  • Z(x,y) represents all the integers between x and y excluding both x and y (x and y are integers with x<y)
  • R(x,∞) represents the set of all numbers greater than x (x is any number)
  • Z(x,∞) represents the set of all integers greater than x (x is any integer)
  • R(-∞,y) represents the set of all numbers less than y (y is any number).
  • Z(-∞,y) represents the set of all integers less than y (y is any integer).
This notation makes it seem like ∞ is a number but it is just a symbol we call “infinity”. With R(x,∞) ∞ means that no finite number we might put in its place will be big enough to describe the entire interval. With R(-∞,y) ∞ means that no finite number we might put in its place will be small enough to describe the entire interval.
Extrapolating a bit from this notation can you find simple ways of representing these intervals Z[1,∞) and R(-∞,∞)??
 

answer

Z[1,∞)=N and R(-∞,∞)=R.
 
When we are done with infinite sets we will understand how R(0,1) is bigger than Z and the same size as R.

Counting

Imagine you are counting apples in a basket. One way to do it is to put labels, 1,2,3,…,n, on them. When the last label is n, that is the size of the set of apples in that basket.
This being a book about mathematics, our labels will be elements of N and we will link those labels in a one-to-one way with the apples instead of putting labels on them. Of course the linking starts with the label 1, and continues with 2, then 3, and so on.

Lines are a good way to show the links between the numeric labels and the set's elements but sometimes they are too imprecise. For more precision we define a link between a set A and a set B as a two-element set with one element from A and the other from B. Each of these elements is an end of the link. A link between A and B is the same as a link between B and A because two element sets {x,y} and {y,x} are the same set.

Otherwise the link between 1 in N and 1 in R would be a one element set (not a two element set with one element in N and the other in R as required). However since all we are doing is finding a way to count elements it is ok to think of N and R as different copies of themselves.

answer

No. Such linkings between N and R are mere failures at counting all the real numbers. To show that there is no way of linking all the natural numbers in N with all the numbers in R requires a clever argument. Proving a negative tends to be difficult.

Matchings

Our basic tool for investigating the relative sizes of sets is the matching. Here are the relevant definitions:
A set of links between a set A and another set B is called a relation. Since the link {a,b} is the same as the link {b,a}, relations are symmetric, i.e. a relation between A and B is also a relation between B and A.

A relation R
  • covers A in case every element of A is included in some link of R.
  • separates A in case every element of A is included in no more than one link of R.
A relation is a matching in case it separates both A and B.

A matching is complete in case it covers both A and B.
Here is an example of a set of links that is not even close to being a matching
because neither A or B is separated or completely covered.
 
Here is a property of matchings we will need in later chapters.
P7.1
Any subset M' of a matching M between A and B is a matching between A and B.
Proof
Since M' is clearly a relation, all we need to know is that M' separates both A and B, i.e. no two links in M' have an end in common. They will not because, if they did, there would be two links in M that have an end in common.
 
There is a subtlety here worth thinking about. The empty set ∅ is a subset of M and so P7.1 says it is a matching. This is ok because the condition that makes a relation a matching only talks about links in the relation. When there are no links, there is no condition to satisfy. The definition of a matching excludes relations with links that share an element. All other relations are accepted.
 
P7.2
A relation R between A and B is a matching exactly when every element of A⋃B is in at most one link of R.
Proof
This result just applies the concept of separation to both A and B.
P7.3
A relation R between A and B is a complete matching exactly when every element of A⋃B is in exactly one link of R.
Proof
This result follows from P7.2 and the fact that a matching R is complete exactly when both A and B are covered.
 
Let's return to thinking about the empty set because that can be tricky. Here is a question to practice on: can you say what has to be true for the empty set to be a complete matching??
 

answer

Since ∅ is always a matching, P7.3 tells us that all we need is for every element of A⋃B to be in exactly one link. So if either A or B has any elements, the empty set, having no links at all, cannot be a complete matching between them. But if both A and B are empty then the empty set is a complete matching because all the elements found in A and B are in exactly one link of ∅.

A Theory of Set Sizes

The theory of set sizes we learned in elementary school has us counting the elements in a finite set until the counting stops. The last number counted is called the cardinality of a set S and it is denoted |S|. For finite sets, cardinality measures size.
The |x| notation has multiple meanings. It means absolute value when x is a number and cardinality when x is a set.
In this monograph the concept of a complete matching replaces the concept of cardinality. We don't count elements of sets unless they are finite and even then we don't have to count elements to apply the concept of a complete matching. Instead of counting, we say two sets A and B are size-equal in case there is a complete matching between them.
So when S is finite, it can be counted, has cardinality |S|, and is also size-equal with Z[1,|S|].
We say a set S is countable in case S is has a complete matching with Z[1,|S|] or with N. We can think of a set as being countable in case it has a complete matching with a subset of N. Countable sets can be counted even if the counting never comes to an end.

Can you argue that a matching of the kind we don't want can be reduced to a matching of the kind we want??
This is not easy to answer with an appropriate amount of detail, but it is worth thinking about before you see my solution.
 
answer
The matchings we don't want have essentially been counted by skipping over some of the required numbers. For example {2,5,6,9} may have been matched to some four element set. Skipping over some of the required numbers means that the complete matching between S and a subset of N has a link {k,s} for which there is no link {k-1,s'} in the matching. Call such a k undesirable. Sum all the undesirable ks together. This sum will be zero exactly when there are no undesirable k.

Pick an undesirable k, remove the link {k,s} and replace it with a link {k-1,s}. What you get is another complete matching between S and a subset of N, but the sum of all ks for this complete matching is one less.

Repeat this process of replacing one link with another until the sum of all ks is 0. That means there are no such ks and the complete matching is between S and Z[1,|S|].

The Mystery of Proper Subsets

We have a seeming paradox. On one hand N and Z are size-equal. On the other N is a proper subset of Z that seems to be much smaller because there are infinitely many integers in Z that are not included in N.
The paradox is only apparent, not real, because we are judging size by two completely different standards. One standard uses complete matchings, the other has us thinking in terms of proper subsets. The reason these two standards seem to be consistent with each other is that they really are consistent with each other—when discussing finite sets.
Thinking terms of proper subsets, we say a subset S' is smaller than a set S when S has elements missing from S'. With finite sets this means when counting S you come to a larger number than if you stopped after first counting S'. But that explanation has no meaning when S is infinite because there is no end to the counting.
If both S and its subset S' are infinite, then neither set will ever be fully counted. This makes things very different as this diagram shows
Every two arrows of counting Z becomes one arrow in counting N. Both sets can have complete matchings with N.
By the way, super countable is not a concept.
 
The basic ideas behind the theory for comparing set sizes presented here were established by the German mathematician, Georg Cantor, who was active in the last part of the 19th century. However Cantor's imagination took him further than we will go. He defined the concept of a cardinal number that expresses the size of a set, finite or infinite. Because numbers don't make much sense without arithmetic, Cantor defined a kind of transfinite arithmetic. As you might expect, when working with infinite cardinalities, transfinite arithmetic is rather weird, but when working with finite cardinalities it is consistent with the arithmetic we already know.

Here we use Cantor's ideas without bothering with cardinal numbers or transfinite arithmetic. A theorem attributed to Ernst Schröder and Felix Bernstein helps make this possible.
