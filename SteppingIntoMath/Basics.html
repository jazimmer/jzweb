Basics

The material in this chapter probably contains a lot of review. However you ought to read it, if only to make sure we are speaking in the same dialect of mathematics.

Sets

You can think of a set as a container if you like. I do. Nevermind the question of what a container is. It is the rules that every set must follow that are important. Two of the most important of these are:
  • We may not talk about something as a set unless we believe that for every other thing there is a definitive answer to the question of whether that other thing is contained in (i.e. an element of) our set.
  • No set can, directly or indirectly, contain itself.
To illustrate the first rule: we may never come in contact with the vast majority of electrons in the cosmos but we know how to recognize electrons and we know a container that gathers together all electrons will not contain itself. Therefore we can talk about the set of all electrons.
The second rule may seem self-evident but experience has shown that we still have to be careful not to violate it, especially indirectly, when we are dealing with sets within sets.
If these rules are not satisfied, what we are talking about isn't the right kind of container to be a set.
The actual rules for working with sets are called axioms. They are written much more symbolically than you will find in this monograph. You can find courses about them in math departments.
 
Things that are contained in a set are called its elements and we write x∊S to say that x is an element of a set S.
Sometimes we describe a set by listing all its elements. This is done with curly brackets like this:
{apple,orange,pear}
Sometimes our description only identifies a set's elements by example:
{1, 2, 3, ...} {1/1, 1/3, 1/5, ...} {1, 2, 3, ...,n}
We can easily recognize the elements of the first two of these sets. The number 3.1415 is not in either of them. The number 1,000,001 is only in the first and the number 1/21 is only in the second. The number 2/15 is not in any of the sets because it fits none of their patterns. When sets are described this way only elements that fit the pattern are included.
Of course there is a potential for ambiguity here. When that is a problem, supporting documentation must clear it up.
The third set is different from the first two in that it is finite. In this context the variable n is inferred to be an integer representing the last listed element of the set. If n≤0, the set is considered to be ∅ (aka the empty set).
 
A more accurate way to describe a set is to explain how to recognize an element:
{ x | x is an electron}
Here's another example. (It makes use of the symbol ∧ which means “and”.)
S = { i | i is an integer ∧ i<0 ∧ i>0 }
It may surprise you that this is a valid description but this set doesn't contain itself and we can always answer the question “is x∊S?”. So S=∅.
While sets cannot contain themselves, they can contain other sets as in
S = { {1,a,α}, {α,γ}, -1, -2 }
A set a A is a subset of a set B in case all elements of A are elements of B and A is a proper subset of B in case B has at least one element that is not in A.
 
So is {1,a,α} a subset of the S defined above??
 

answer:

No, {1,a,α}∊S while 1, a, and α are elements of {1,a,α}. Other elements of S are
{α,γ} -1 -2
 
Given two sets A and B some commonly used notations are
  • A⊆B which means A is a subset (not necessarily proper) of B
  • A⋃B which means the set of all elements in either A or B.
  • A⋂B which means the set of all elements in both of A and B
  • A-B which means the set of all elements in A that are not in B.
Sets A and B are disjoint in case A⋂B=∅, i.e. they have no element in common.
If S is a set of sets, we can use the ⋃ symbol to represent all the elements in all of them.
⋃S = { x | x∊Y for some Y∊S }
For an infinite set of sets S, it is not always obvious that ⋃S exists. A common way around this is to assume an axiom called the Axiom of Union.
 
If
S = {{a,b},{c,d,e},{b},{e,f}}
what is ⋃S??
 

answer:

{ a,b,c,d,e,f }

One could write {a,b,b,c,d,e,e,f} but that would be bad form since
{a,b,b,c,d,e,e,f} = {a,b,c,d,e,f}
All we know about b in either of the above sets is that it is there. There is no concept of it being their in multiple copies.
 
A partition of a set S is a set of nonempty subsets P of S such that ⋃P=S and every two distinct sets A and B in P are disjoint.
It is understood that if we talk about two arbitrary elements x and y of some set S, then they may be the same element. If we want to say they are different, then we say they are distinct.
 
These ideas will be important later. Let's play with them. Can you partition {a,b,c,d,e,f} into two element subsets, one of which is {a,f}??
 

answer:

Yes. One way to do this is
{ {a,f}, {b,c}, {d,e} }
 
Here is the start of an alternate definition of a partition
A partition of a set S is a set of nonempty subsets P of S such that ⋃P=S and if any two sets in P have an element in common then … .
 
Without changing what is written, can you finish the sentence so that the definition is equivalent to the one above??
 

answer:

A partition of a set S is a set of nonempty subsets P of S such that ⋃P=S and if any two sets in P have an element in common then they are the same set.

Tuples

Tuples exist so that we can have multiple copies of an element in a container. Tuples list their elements in a specific order. There are a couple consequences of giving each element a position:
  • Tuples can contain multiple copies of the same element.
  • Rearranging the elements of a tuple creates a different tuple.
So these are all different tuples
(a,b,c,a) (c,b,a,a) (b,a,c)
The first two of these tuples are 4-tuples, the third is a 3-tuple. When a tuple has just two values, as in (x,y), it is called a pair. What is the difference between the following??
  1. x,y∊S
  2. (x,y)∊S
 

answer

The first says both x and y are element of S and the second says that the pair (x,y) is an element of S.

Variables

Variables name things. We may have a friend named “Bob”. We use that name to refer to him, but the name has an existence of its own, and in other contexts it may refer to other people. Or we may know the name “Bob” without knowing anybody with that name. (Is there a Bob here?)
So it is with variables. They can refer to a particular value in one context and another value in a different context.
When the value of a variable is unknown we have a known unknown on our hands. An analogous example in real life is looking for a criminal named “John Doe”.
When we say that a variable x
  • is assigned a value, we mean that x is a name for that value—until the context changes
  • is arbitrary, we mean it is a known unknown.
  • depends on other variables, we mean the value it refers to can be deduced when we know what all those other variables refer to.
  • can have one of two values, we mean that x's value, while unknown, has been narrowed down to two possibilities.

Numbers

We all have been taught to visualize our number system on a line like this:
There are no breaks in this line. That means for any two numbers n1 and n2 with n1<n2 there will be another number n3 such that n1<n3<n2. This is true no matter how close n2 is to n1. This is called a density property.
Numbers like 27.23 are called “real numbers”. I will use R to refer to the set of all of them. Usually when I say “number” I mean a real number.
Two subsets of R are important for us
  • The natural numbers
    N = {1,2,3,…}.
    The integers
    Z = {…,-3,-2,-1,0,1,2,3,…}.
Although N and Z subsets of the real numbers, we will often think of them as separate sets.
 
Can you fill in this definition of a set P so that it partitions the integers into positive numbers, negative numbers, and 0??
{ { … | … }, {…}, { … | …} }
 

answer:

This would work but it doesn't use the { … : …} notation.
{{1,2,3,…},{0},{-1,-2,-3,…}}
Here are two better answers
{ N, {0}, { -i | i∊N } }
and
{ { i | i∊Z ∧ i>0 }, {0}, { i | i∊Z ∧ i<0 } }
 
Here are some things about numbers you may have forgotten.
  • All numbers can be expressed with infinitely many fractional digits.
  • When those fractional digits end with an unending string of 0s as in 1.000… or as in 0.125000…, the 0s at the end can be left off as in 1 or 0.125.
  • A number like a 0.999… or 2.3499… (i.e. one that ends with an unending string of 9s) isn't a separate number. Instead these numbers are the same as the next larger number, i.e. 0.999… is 1 and 2.3499… is 2.35. This isn't rounding. It's a quirk of the way we write down numbers.
  • Some fractions are unfinished whenever we write them down with fractional digits—no matter how many fractional digits we write down, we are expressing an approximation. For example 1/3 is not exactly 0.333…3, no matter how many fractional digits we write down.
  • For some of the unfinished numbers (e.g. π) we cannot even predict what all the factional digits will look like. (You may have learned the name “irrational” to describe such numbers.)
 
Given two numbers n1 and n2 such that n1<n2. Can you show that, no matter how close together n1 and n2 are, there is an infinite sequence of numbers c1,c2,c3,… that lie strictly between n1 and n2??
Think of this as a puzzle. And try to forget any incantations you learned in math class. Simply use the density property to explain a process for generating infinitely many numbers between n1 and n2.
 

answer:

  • Choose c1 such that n1<c1<n2 using the density property.
  • Similarly choose c2 such that c1<c2<n2. (Therefore n1<c2<n2.)
  • Choose c3 such that c2<c3<n2. (Therefore n1<c3<n2.)
  • Continue choosing numbers this way getting an unending sequence of numbers between n1 and n2.
 
A math major is likely to take an entire semester course on the development of our number system. Here I skip over that development and rely on our common understanding so you can read examples of mathematical reasoning as applied to topics outside the usual mathematics major.

Sequences

A sequence is rather like a tuple. In both cases there will be a first element, a second element, and so on. But sequences are written without parentheses and there may be infinitely many elements in them.
So
(1,2,1,3,2,4,3)
is a tuple but these are sequences
1,2,1,3,2,4,3 1,2,3,…,n 1,2,4,8,…
The first of these sequences has 7 elements, the second has n elements where the actual number of elements is not known, and the third has infinitely many elements.
The last two of these sequences are written with an ellipse (…). When that is done, the sequence ought to exhibit a pattern that we can use to predict the missing elements. For the first and second sequences, the pattern is the next element is always one larger. For the third sequence, the next element is two times its predecessor.
Traditionally mathematicians start counting sequence elements with 1. However with the influence of computer programming, counting may start with 0. In this monograph though, we always start counting with 1.

Index Sets

Some sequences contain known unknowns and so their many values have to be represented with separate variables. This is done with subscripts as in
w1, w2, w3, …
The numbers are indexes or indices and they indicate the order of the elements in the sequence
Of course if we put the same variables in a set
{w1, w2, w3, …}
the order becomes unimportant. Knowing the order is unimportant we are no longer restricted to using N for our index set. Any set S will do, as in
{ wi | i∊S }
